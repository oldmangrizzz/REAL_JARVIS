EXECUTION HARNESS The self-healing harness is MetaHarness. It parses the Archon workflow YAML into a DAG of planning, implementation, review, and any added repair nodes. It then loads execution traces, identifies failure hotspots, diagnoses common causes such as missing dependencies, schema mismatches, or validation failures, and mutates the workflow in place. If validation is missing, it inserts a validation node that runs xcodebuild against the Jarvis project. If review is not gated on validation, it rewires that dependency. If a dependency hotspot is detected, it appends a counterfactual-diagnosis node to inspect raw traces and repair the failing branch. Every mutation is hashed for rollback and scored with a simple evaluation function based on success count, failure penalty, and mutation bonus. Observability is split cleanly: TelemetryStore writes JSONL records for execution_traces, stigmergic_signals, recursive_thoughts, and harness_mutations, while Convex mirrors the same operational surfaces in strongly typed schemas. That gives the host tunnel and control plane a real-time view of recursive thought chains, pheromone movement, and workflow rewrites.
