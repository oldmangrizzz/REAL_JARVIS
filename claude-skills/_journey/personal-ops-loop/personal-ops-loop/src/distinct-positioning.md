# Distinct Positioning

Use this note when publishing or describing the kit.

## What to emphasize

- This is a **behavioral governance kit** for personal agents.
- It focuses on **restraint**, not maximum automation.
- It solves the mundane but important problems: duplicate reminders, privacy boundaries, quiet hours, and social judgment.

## What to avoid claiming

Avoid positioning it as:
- a generic assistant starter
- a memory engine
- a full automation stack
- an inbox pipeline
- an autonomous daemon loop

## Short positioning lines

- A behavioral operating model for personal agents.
- A safety-first kit for proactive assistants with judgment.
- A reusable pattern for bounded personal-agent behavior.

## Sharp differentiators

- stateful reminders instead of repetitive nagging
- private/shared memory separation instead of flat memory dumping
- presence-aware nudges instead of sensor spam
- quiet hours and channel-sensitive behavior instead of constant output
- approval gates for outbound actions instead of silent execution
